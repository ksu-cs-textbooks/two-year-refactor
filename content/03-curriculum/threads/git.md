## Character

Redesigned from a single late pass into four. Version control as 'tracking how things change over time' is introduced as a READING tool first, before it is a workflow tool.

## Passes across the two years

| Block | Host context | What’s new at this pass |
|---|---|---|
| B1 (Y1 Fall) | CS-101/102 (embedded) | Git as a history-reading tool: log, diff, blame; semantic versioning as “what a version communicates.” Not yet a workflow. |
| B2 (Y1 Fall) | CS-105 Code Reading & Repair | Commit and revert as a safety net for repair work. |
| B5 (Y2 Fall) | CS-202 (embedded) | Solo feature branching — try a schema alternative safely; branching as a solo skill before a team skill. |
| B6 (Y2 Fall) | CS-205 Collaborative Development | Collaborative Git: PRs, code review, merge conflicts with other people’s changes. |
