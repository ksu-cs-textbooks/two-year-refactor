## CS-206 — Collaborative Development

**1 credit**

### Working description

Collaborative Git (branches, PRs, code review, conflict resolution); team practices. Conway's Law enacted — the team is now an organization, and how it divides work shapes what it builds. Signature assessment: Team Software Project begins, carrying a responsibility retrospective (correctness, safety, teamwork).

## Competency Assessments

**Signature assessment begins here:** Team Software Project — carries through Block 8 (CS-211). The Block 6 pass anchors the responsibility retrospective (evidence generated: Collaborate on Software Teams; Implement Software Solutions; Verify Software Quality; Maintain Software Systems; Evaluate Ethical Implications; Reflect on Professional Growth).

| Assessment type | This pass | Competencies evidenced |
|---|---|---|
| Code Archaeology | Git history archaeology — read a PR or commit history, explain what changed and why | Analyze Computational Systems; Maintain Software Systems |
| Implementation Task | Feature implementation in a collaborative codebase — professional workflows, team-defined specs | Implement Software Solutions |
| Maintenance Task | Feature branch PR — implement a change in a shared codebase, pass code review | Maintain Software Systems |
| Debugging Challenge | Code review — find latent bugs in a peer's code before they manifest | Debug Software Systems |
| Environment Configuration Task | Configure a shared team environment — branch protection, PR templates, shared linting config | Manage Computing Environments |
| Requirements Analysis Task | Stakeholder interview + requirements brief — conduct an interview, produce a documented requirements artifact with acceptance criteria | Gather and Analyze Requirements |
| Ethical Evaluation | Responsibility retrospective — identify system threats and team ethical responsibilities | Evaluate Ethical Implications |
| Technical Explanation Task | PR description and code review — communicate what changed, why, and what to look for | Communicate Technical Information |
| Collaboration Task | Full collaborative PR cycle — branch, implement, open PR, respond to review, merge | Collaborate on Software Teams |
| Reflection Assessment | Responsibility retrospective — team reflection on ethical and professional practice | Reflect on Professional Growth |
