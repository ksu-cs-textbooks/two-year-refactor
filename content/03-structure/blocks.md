## Representation and Reading
What is a computational system?
How is information represented?
How do I understand what software is doing?

Major threads:
- Information representation
- Program representation
- State and Memory
- Reading computational artifacts
- Process & Execution
- Problem decomposition

Math connections:
1. Sets and classification
2. Logic
3. Boolean reasoning
4. Functions
5. State

Studio experiences:
- Computational artifact investigation
- Trace and predict
- Data explorer
- Program modification

Place HTML/CSS alongside CSV, JSON, Markdown, configuration files, and source code as examples of **computational artifacts that students learn to read, explain, and modify**.
